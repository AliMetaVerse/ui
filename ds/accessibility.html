Accessibility

Guidelines and best practices for creating inclusive and accessible user experiences.

Accessibility Principles

Our design system follows the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level, ensuring our products are usable by people with a wide range of abilities. We're committed to creating inclusive experiences through these core principles:

Perceivable

Information and user interface components must be presentable to users in ways they can perceive.

  • Provide text alternatives for non-text content
  • Create content that can be presented in different ways
  • Make it easier for users to see and hear content

Operable

User interface components and navigation must be operable by all users.

  • Make all functionality available from a keyboard
  • Give users enough time to read and use content
  • Help users navigate and find content

Understandable

Information and operation of the user interface must be understandable.

  • Make text readable and understandable
  • Make content appear and operate in predictable ways
  • Help users avoid and correct mistakes

Robust

Content must be robust enough to be interpreted by a wide variety of user agents, including assistive technologies.

  • Maximize compatibility with current and future user tools
  • Use semantic HTML elements appropriately
  • Ensure proper ARIA attributes when needed

Color Contrast

Sufficient color contrast between text and background ensures readability for all users, including those with visual impairments.

Text Contrast Requirements

  • Normal text: Minimum contrast ratio of 4.5:1
  • Large text: Minimum contrast ratio of 3:1
  • UI components and graphics: Minimum contrast ratio of 3:1

Keyboard Navigation

All interactive elements must be accessible via keyboard navigation to accommodate users who cannot use a mouse or other pointing device.

Focus Indicators

All interactive elements must have a visible focus indicator with sufficient contrast. Never remove focus outlines without providing an alternative.

Key Focus Styles

  • 3px solid outline in primary color
  • 2px offset from the element edge
  • Minimum 3:1 contrast ratio against adjacent colors

Semantic HTML & ARIA

Using semantic HTML elements and appropriate ARIA attributes ensures that content is properly interpreted by assistive technologies.

Semantic HTML Best Practices

  • Use appropriate heading levels (<h1> through <h6>) to create a logical document structure
  • Use <button> for clickable actions, not <div> or <span>
  • Use <a> elements for navigation to new pages or resources
  • Use <ul>, <ol>, and <li> for lists
  • Use proper form elements with associated labels

Common ARIA Attributes

ARIA attributes should be used to enhance accessibility when HTML semantics are not sufficient. Always prefer native HTML semantics when possible.

ARIA Attribute Purpose
aria-label Provides an accessible name for elements with no visible text
aria-labelledby Associates an element with text that labels it
aria-describedby Associates an element with text that describes it
aria-expanded Indicates if a control is expanded or collapsed
aria-hidden Hides content from assistive technologies

Accessibility Checklist

Use this checklist when implementing components to ensure they meet our accessibility standards:

Content & Structure

  • Page has a clear, descriptive title
  • Headings create a logical document outline
  • Images have meaningful alt text
  • Text can be resized up to 200% without loss of content
  • Content is understandable without reliance on color

Interactive Elements

  • All interactive elements are keyboard accessible
  • Focus order is logical and intuitive
  • Focus indicators are clearly visible
  • Interactive elements have appropriate hit areas (min. 44Ã—44px)
  • Form elements have associated labels

Visual Design

  • Text meets minimum contrast requirements
  • UI components meet minimum contrast requirements
  • Content is readable in various screen sizes
  • Design works in both portrait and landscape orientation
  • Content is usable with zoom set to 400%

Advanced Techniques

  • Custom components use appropriate ARIA roles/attributes
  • Dynamic content changes announce updates to screen readers
  • Non-standard interactions have clear instructions
  • Animations can be paused or disabled
  • Media has appropriate controls and captions
