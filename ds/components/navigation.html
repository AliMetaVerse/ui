Navigation Components

A collection of components for user interface navigation across applications.

Tab Navigation

Tabs organize content into different sections, with only one section visible at a time.

Overview
Analytics
Reports
Settings

Content for the Overview tab panel is displayed here.

Usage Guidelines

  • Use tabs to organize content that belongs to the same context but can be clearly separated.
  • Label tabs with short, clear text (1-2 words) that describes the content within.
  • Maintain the same tab order across different pages for consistency.
  • Highlight the active tab clearly to indicate the current view.
  • Limit the number of tabs to 5-7 to avoid overwhelming users.

Code Example

<div class="nav-tabs">
  <div class="nav-tab active">Overview</div>
  <div class="nav-tab">Analytics</div>
  <div class="nav-tab">Reports</div>
  <div class="nav-tab">Settings</div>
</div>
<div class="tab-content">
  <div class="tab-pane active">
    <p>Content for the Overview tab panel...</p>
  </div>
</div>

Breadcrumbs

Breadcrumbs show the hierarchical path to the current page and provide navigation to parent pages.

Home
Surveys
Employee Satisfaction Survey

Usage Guidelines

  • Use breadcrumbs for interfaces with hierarchical navigation and multiple levels.
  • Place breadcrumbs near the top of the page, below the main navigation.
  • Keep breadcrumb labels concise and descriptive.
  • Make each breadcrumb item a clickable link, except for the current page.
  • Use a clear separator between breadcrumb items.

Code Example

<div class="breadcrumbs">
  <div class="breadcrumb-item">
    <a href="#">Home</a>
  </div>
  <div class="breadcrumb-separator">
    <svg>...</svg>
  </div>
  <div class="breadcrumb-item">
    <a href="#">Surveys</a>
  </div>
  <div class="breadcrumb-separator">
    <svg>...</svg>
  </div>
  <div class="breadcrumb-item active">
    Employee Satisfaction Survey
  </div>
</div>

Pagination

Pagination divides content into separate pages and provides navigation between them.

1
2
3
4
...
10

Usage Guidelines

  • Use pagination when displaying large sets of data or content items.
  • Show the current page clearly with proper styling.
  • Include previous/next controls for easy sequential navigation.
  • Use ellipsis (...) to indicate skipped pages when there are many pages.
  • Consider showing the total number of pages or items for context.
  • For smaller datasets, infinite scrolling or "load more" might be more appropriate.

Code Example

<div class="pagination">
  <div class="pagination-item"><svg>...</svg></div>
  <div class="pagination-item">1</div>
  <div class="pagination-item active">2</div>
  <div class="pagination-item">3</div>
  <div class="pagination-item">4</div>
  <div class="pagination-item">...</div>
  <div class="pagination-item">10</div>
  <div class="pagination-item"><svg>...</svg></div>
</div>

Dropdown Menu

Dropdown menus display a list of options that appear when triggered by a button.

Edit Survey
Duplicate
Share
Export Results
Delete

Usage Guidelines

  • Use dropdowns to conserve space when there are multiple related actions or options.
  • Label the dropdown button clearly to indicate what options it contains.
  • Include a visual indicator (like a chevron) to show that it's a dropdown.
  • Close the dropdown when a user clicks outside it or selects an option.
  • Group related actions and consider using dividers for different categories.
  • Place destructive actions (like Delete) at the bottom and highlight them appropriately.

Code Example

<div class="dropdown">
  <button class="dropdown-toggle">
    <span>Actions</span>
    <svg>...</svg>
  </button>
  <div class="dropdown-menu">
    <div class="dropdown-item">Edit Survey</div>
    <div class="dropdown-item">Duplicate</div>
    <div class="dropdown-item">Share</div>
    <div class="dropdown-item">Export Results</div>
    <div class="dropdown-item text-error">Delete</div>
  </div>
</div>

Sidebar Navigation

Sidebar navigation provides vertical menu options for navigating between different sections or pages.

Dashboard
Surveys
Reports
Analytics
Settings

Usage Guidelines

  • Use sidebar navigation for applications with multiple sections or pages.
  • Highlight the active item to show the current location.
  • Consider collapsible sections for complex navigation structures.
  • Ensure the sidebar is responsive and adapts to different screen sizes.
  • Use consistent icons and labels for easy recognition.
  • Consider a collapsible/expandable sidebar for better space utilization.

Code Example

<div class="sidebar-nav">
  <div class="sidebar-nav-item active">Dashboard</div>
  <div class="sidebar-nav-item">Surveys</div>
  <div class="sidebar-nav-item">Reports</div>
  <div class="sidebar-nav-item">Analytics</div>
  <div class="sidebar-nav-item">Settings</div>
</div>

Accessibility Considerations

Keyboard Navigation

  • Tab focus: Ensure all interactive navigation elements can be accessed with keyboard tabbing.
  • Arrow keys: Support arrow key navigation within components like tabs and dropdown menus.
  • Enter/Space: Allow activation of navigation elements with Enter or Space keys.
  • Escape: Close dropdowns and menus when the Escape key is pressed.
  • Focus indicators: Provide visible focus states for all interactive elements.

Screen Reader Support

  • ARIA roles: Use appropriate ARIA roles (e.g., role="navigation", role="menu").
  • ARIA states: Include states like aria-expanded, aria-current, and aria-selected.
  • Hidden content: Use aria-hidden="true" for decorative elements.
  • Clear labels: Provide descriptive labels for navigation landmarks.
  • Skip links: Include skip navigation links for keyboard users.
