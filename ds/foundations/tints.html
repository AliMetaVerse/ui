Understanding Tints & Shades

Our color system utilizes a scale of tints and shades to ensure consistency and accessibility across the UI. Each color in our palette has variations ranging from 50 (lightest) to 900 (darkest).

Tints

Tints are lighter variations of a color, created by adding white. We use tints for backgrounds, hover states, and subtle UI elements.

Base Colors

Base colors (500 level) are the core brand colors that define our visual identity. These are used for primary UI elements.

Shades

Shades are darker variations of a color, created by adding black. We use shades for text, borders, and for emphasis.

Best Practices

Do

  • Use colors consistently across the interface to maintain visual harmony
  • Use light tints (50-200) for backgrounds and subtle UI elements
  • Use mid-range tints/shades (400-600) for interactive elements and emphasis
  • Use darker shades (700-900) for text to ensure readability
  • Always check color contrast for accessibility compliance
  • Use semantic colors for their intended purposes (success, error, warning, info)

Don't

  • Don't use too many colors in a single view
  • Avoid using colors in ways that contradict their semantic meaning
  • Don't use light text on light backgrounds or dark text on dark backgrounds
  • Avoid relying solely on color to convey meaning (for color-blind users)
  • Don't create custom color variations outside the defined palette
  • Avoid using primary colors for large background areas
