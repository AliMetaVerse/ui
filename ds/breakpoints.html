Breakpoints

Responsive breakpoints for adapting layouts to different screen sizes.

Breakpoint System

Our design system uses Tailwind CSS breakpoints that follow common device widths. These breakpoints help create responsive layouts that work well across different screen sizes.

Extra Small (xs): Default - Less than 640px
Small (sm): 640px and larger
Medium (md): 768px and larger
Large (lg): 1024px and larger
Extra Large (xl): 1280px and larger
2X Large (2xl): 1536px and larger

Resize your browser to see the active breakpoint change

Breakpoint Prefix Minimum Width CSS Media Query Typical Devices
(none) 0px @media (min-width: 0px) { ... } Mobile phones (portrait)
sm 640px @media (min-width: 640px) { ... } Mobile phones (landscape), small tablets
md 768px @media (min-width: 768px) { ... } Tablets (portrait)
lg 1024px @media (min-width: 1024px) { ... } Tablets (landscape), small laptops
xl 1280px @media (min-width: 1280px) { ... } Laptops, desktops
2xl 1536px @media (min-width: 1536px) { ... } Large desktop screens

Using Breakpoints

Apply responsive designs by prefixing Tailwind utility classes with breakpoint prefixes. This enables different styles at different screen sizes.

<div class="text-sm sm:text-base md:text-lg lg:text-xl xl:text-2xl">
  This text will change size at different breakpoints
</div>

<div class="flex flex-col md:flex-row">
  Changes from vertical to horizontal layout on md screens and larger
</div>

<div class="w-full sm:w-1/2 lg:w-1/3">
  Full width on mobile, half width on tablets, one-third on desktops
</div>

Container Behavior

Our container class sets maximum widths at each breakpoint for consistent layouts:

Breakpoint Container Max-Width CSS
Default (xs) 100% max-width: 100%;
sm (640px+) 640px max-width: 640px;
md (768px+) 768px max-width: 768px;
lg (1024px+) 1024px max-width: 1024px;
xl (1280px+) 1280px max-width: 1280px;
2xl (1536px+) 1536px max-width: 1536px; 
<div class="container mx-auto px-4">
  This container will adapt its maximum width at different screen sizes
</div>

Best Practices

  • Design for mobile first, then enhance for larger screens
  • Test designs at all breakpoints to ensure a smooth experience
  • Use relative units (rem, em, %) instead of pixels when possible for better scaling
